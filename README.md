# Регламент
Каждая ветка, сливающаяся в main должна называться `<номер задачи в ветке>-<название задачи>`, один МР - одна задача (если получится все же вести еще и issues)

# SQL
Файл для бд называется - `YYYY-MM-DD <Что в скрипте делается>.sql`

# Redis
Сделан локально в контейнере, чтобы удобно смотреть чего валяется в нем можно юзать Redis Insight

# DevOps
В CI части только линтер (можно будет и притир докинуть, но он на беке есть, как и сам линт без джобы)
Часть delivery - собираются образы, кидаются на докерхаб
CD - деплой на виртуалке, в скрипте подключается к виртуалке и запускает скрипт на ней deploy, в котором скачиваются образы с докерхаба нужные, также передается тег на виртуалку, вид скрипт "deploy <тег>"
